AUDITORÍA FJORD VI 1.3.9

Base auditada:
- Fjord_VI_1_3_8_ADMIN_U_UNDEFINED_FIX.zip

Resultado general:
- Sintaxis Python: OK
- Templates Jinja: OK
- Admin u undefined: corregido
- HTML suelto después de </html>: no detectado
- Ojos de clave inline: conservados desde 1.3.7/1.3.8
- Cambio obligatorio de clave inicial: presente
- Reset administrativo visible: corregido en 1.3.9

Hallazgo importante:
El botón visible de reset en Administración usaba /admin/reset_password/{u.id}.
Esa ruta reseteaba la clave a demo1234, pero no marcaba must_change_password=True.
Eso dejaba la clave temporal funcionando sin obligar al usuario a crear clave personal.

Corrección aplicada:
- target.password_hash = hash_password("demo1234")
- target.must_change_password = True
- retorno a /admin?page=socios&msg=clave_reseteada
- log actualizado como reset de clave temporal con cambio obligatorio

Conclusión:
La versión 1.3.9 es más consistente que 1.3.8 para producción, porque cierra el ciclo completo:
1. usuario nuevo debe crear clave;
2. usuario con demo1234 debe crear clave;
3. usuario reseteado por Administración debe crear clave;
4. cambio voluntario funciona separado;
5. Admin vuelve a renderizar sin error de u indefinida.
